O que é git?
-historico de projeto
Linha do tempo


	---conceitos basicos---

-Repository
Local onde ficará o histórico do seu projeto

-Branch
Linha do tempo

-Commit
Pontos na história

-Stage
Preparação do que será enviado para o ponto na história



//inicia o git(repositório) no seu projeto
	git init

//adiciona todos os arquivos modificados ao stage
	git add .(esse pontinho adiciona todos os arquivos modificados que estão nesse comando, na
pasta).

//cria e descreve ponto na história
	git commit -m "message here"

//envia alterações para o repositório remoto
	git push





	---passos pra criar um githug----

1- passo iniciar o Terminal no vscode e ir em New Terminal
2- digitar git -v para ver a versão do seu git
3- digite git init o vscode vai mudar as cores porque diz que os arquivos sao novos e nao estao
na STAGE AREA
4- git add .   vai colocar todos os arquivos na STAGE AREA e vai mudar a cor dos arquivos para
verde claro.
5- git commit -m "initial commit"(nessas aspas voce pode colocar qualquer mensagem)     esse 
-m(message) ou qlqr outra letra, é chamado de flag ou ar
gumento)  as cores mudaram para branco igual estava antes de começar o Terminal.
6- git push   para enviar para o github
